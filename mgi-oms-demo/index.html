IF Product line = Floor Drains, Cleanout Drains, Roof Drains, Trench Drains → Route to EIS (Smith Drain)
R3
IF Order contains items from multiple divisions → Split Order Route each line to correct ERP
R4
IF Fulfillment location flagged as delayed (>2 days) → Reroute Alert rep + suggest alternate
R5
IF New division / ERP added to MGI infrastructure in future → Add Rule No ERP config change required
Why not just give reps SAP access?
SAP access is ERP-specific. Reps would still need separate access to EIS/Progress for Smith Drain. Two logins, no unified view.
SAP doesn't expose natively to Experience Cloud communities. Building a rep portal out of SAP is a heavier integration than OMS + Thomas's existing community POC.
SAP doesn't give you the agentic WISMO layer. The "where's my order" conversation still ends in a phone call.
The bottom line for MGI
OMS extends the Salesforce infrastructure MGI already owns — to solve a problem SAP alone can't.
✓One rep portal — all orders, both ERPs, no double entry
✓Agentic order interaction — reps get answers without calling in
✓SAP + EIS unchanged — nothing ripped, nothing replaced
✓Thomas's team works in Salesforce — no SAP development required
✓Future-proof — add more ERPs or divisions with a new routing rule
